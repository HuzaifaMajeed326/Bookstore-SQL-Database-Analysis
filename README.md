# 📚 Bookstore SQL Database Analysis

## 📌 Project Overview
This project demonstrates SQL database design and data analysis using a Bookstore Management System. The database was built in PostgreSQL and includes tables for books, customers, and orders. Various SQL queries were written to retrieve business insights from the data.

---

## 🎯 Objectives

- Design a relational database
- Create tables with primary and foreign keys
- Import CSV datasets
- Perform data analysis using SQL
- Practice joins, aggregate functions, filtering, grouping, and sorting

---

## 🗂 Database Schema

The project contains three tables:

### Books
- Book_ID
- Title
- Author
- Genre
- Published_Year
- Price
- Stock

### Customers
- Customer_ID
- Name
- Email
- Phone
- City
- Country

### Orders
- Order_ID
- Customer_ID
- Book_ID
- Order_Date
- Quantity
- Total_Amount

## 📈 Business Questions Solved

### Basic Queries

- Retrieve books by genre
- Find books published after 1950
- List customers from Pakistan
- Show October 2023 orders
- Calculate total stock
- Find the five most expensive books
- Find customers ordering more than three books
- Retrieve high-value orders
- List all available genres
- Find the book with the lowest stock
- Calculate total revenue

### Advanced Queries

- Total books sold by genre
- Average price of Hadith books
- Customers with at least three orders
- Most frequently ordered book
- Top expensive Islamic books
- Quantity sold by author
- Cities of high-spending customers
- Top two customers by spending
- Remaining stock after fulfilling orders
